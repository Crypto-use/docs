# Autenticação

A API CryptoUse utiliza autenticação via **Bearer Token** para validar todas as requisições. Para acessar a API, você deve gerar um token de acesso através do painel administrativo.

## Obtendo seu Bearer Token

1. Acesse o [Painel CryptoUse](https://cryptouse.com.br/profile)
2. Navegue até a seção **"API/Webhooks"**
3. Clique em **"Gerar Token"** ou **"Adicionar API"**
4. Copie seu token e armazene em um local seguro. Este token será utilizado nas autenticações.

## Como utilizar o Bearer Token

Inclua o token no cabeçalho HTTP `Authorization` em todas as requisições, no seguinte formato:

```bash
curl -X POST "https://api.cryptouse.com.br/api/v1/onchain/deposit" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer seu_token_aqui" \
  -d '{
    "value": 100.00,
    "asset": "USDT",
    "network": "bsc"
  }'
```

## Ambiente de desenvolvimento

Para testes em ambiente de desenvolvimento, você pode gerar chaves de API específicas para desenvolvimento que não processam transações reais:

1. No dashboard, vá para "Configurações" > "API"
2. Selecione "Ambiente de Desenvolvimento"
3. Clique em "Gerar nova chave de API de teste"

As chaves de desenvolvimento funcionam exatamente como as chaves de produção, mas operam em um ambiente sandbox separado.

## Segurança

- Nunca compartilhe sua chave de API
- Não inclua sua chave de API diretamente no código-fonte que será compartilhado ou em repositórios públicos
- Configure variáveis de ambiente ou cofres seguros para armazenar suas chaves
- Se suspeitar que sua chave foi comprometida, revogue-a imediatamente no dashboard e gere uma nova
