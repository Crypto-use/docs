## Criar solicitação de saque PIX

Este endpoint processa uma solicitação de saque via PIX para a chave PIX fornecida.

### Requisição

<ParamField body="value" type="number" required>
  O valor do saque em BRL
</ParamField>

<ParamField body="pixKey" type="string">
  Chave PIX para o saque. Se não fornecida, será utilizada a chave PIX registrada do comerciante
</ParamField>

### Exemplo de requisição

```bash
curl -X POST "https://api.cryptouse.com.br/api/v1/order/pix-out" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer seu_token_aqui" \
  -d '{
    "value": 100.00,
    "pixKey": "email@exemplo.com"
  }'
```

### Resposta

<ResponseField name="message" type="string">
  Mensagem de sucesso
</ResponseField>

<ResponseField name="amount" type="number">
  O valor sacado
</ResponseField>

### Exemplo de resposta

<CodeGroup>

```json Sucesso (201)
{
  "message": "Saque processado com sucesso",
  "amount": 100.00
}
```

```json Requisição inválida (400)
{
  "message": "Dados inválidos na requisição",
  "error": {
    "details": "Valor de saque inválido"
  }
}
```

```json Não autorizado (401)
{
  "message": "Chave de API inválida ou não autorizada"
}
```

```json Erro interno (500)
{
  "message": "Ocorreu um erro ao processar sua solicitação",
  "error": {
    "details": "Erro ao processar o saque PIX"
  }
}
```

</CodeGroup>

## Limites de saque

Os limites de saque via PIX são:

| Tipo de conta | Limite diário | Limite por transação |
|---------------|---------------|---------------------|
| Padrão        | R$ 5.000      | R$ 1.000            |
| Verificada    | R$ 50.000     | R$ 10.000           |
| Empresarial   | R$ 100.000    | R$ 25.000           |

## Horário de processamento

Os saques PIX são processados 24/7, incluindo finais de semana e feriados. O tempo médio de processamento é de:

- Horário comercial (seg-sex, 9h-18h): até 5 minutos
- Fora do horário comercial: até 15 minutos

## Chaves PIX suportadas

O endpoint aceita todos os tipos de chaves PIX:

- CPF/CNPJ
- Email
- Telefone
- Chave aleatória
- Chave EVP

## Notas importantes

- O saldo disponível deve ser suficiente para cobrir o valor do saque
- A chave PIX deve estar ativa e válida no sistema bancário
- Saques para contas de terceiros podem exigir verificação adicional
- Em caso de falha, o valor é automaticamente estornado para sua conta
