# Introdução à API CryptoUse

A API CryptoUse oferece uma interface completa para integrar pagamentos em criptomoedas e conversão para PIX em seu aplicativo ou site. Nossa API é RESTful, baseada em JSON, e é projetada para ser simples e intuitiva.

## Autenticação

Todas as requisições à API devem incluir sua chave de API no cabeçalho HTTP `X-App-ID`. Veja mais detalhes na seção [Autenticação](/essentials/autenticacao).

## URL Base

```
https://api.cryptouse.com.br
```

## Grupos de Endpoints

Nossa API está organizada em três grandes grupos:

### Onchain

Endpoints para operações diretas em blockchain, como depósitos e saques.

| Método | Endpoint | Descrição |
|--------|----------|-----------|
| POST | `/api/onchain/deposit` | Criar uma ordem de depósito |
| POST | `/api/onchain/withdraw/initiate` | Iniciar um saque |
| GET | `/api/onchain/order/{id}` | Consultar ordem onchain |

### Application Order

Endpoints para criar e gerenciar ordens de pagamento em diferentes métodos.

| Método | Endpoint | Descrição |
|--------|----------|-----------|
| POST | `/api/order/create` | Criar ordem Binance |
| POST | `/api/order/create/lightning` | Criar ordem Lightning |
| POST | `/api/order/create/pix` | Criar ordem PIX |
| GET | `/api/order/{id}` | Consultar ordem |

### Crypto Pix

Endpoints para converter criptomoedas para pagamentos PIX.

| Método | Endpoint | Descrição |
|--------|----------|-----------|
| POST | `/api/crypto-pix/onchain/pay` | Processar pagamento crypto → PIX |
| POST | `/api/crypto-pix/order/get` | Consultar pagamento |

## Formatos de Resposta

Todas as respostas da API são retornadas no formato JSON. Os códigos de status HTTP são usados para indicar o resultado da operação:

| Código | Significado |
|--------|-------------|
| 200 | Sucesso na operação GET ou POST |
| 201 | Recurso criado com sucesso |
| 400 | Erro na requisição ou dados inválidos |
| 401 | Autenticação inválida ou ausente |
| 404 | Recurso não encontrado |
| 500 | Erro interno do servidor |
