## Obter detalhes de uma ordem onchain

Este endpoint permite recuperar informações sobre uma ordem blockchain específica usando seu ID de link de pagamento.

### Parâmetros de URL

<ParamField path="id" type="string" required>
  ID do link de pagamento da ordem
</ParamField>

### Exemplo de requisição

```bash
curl -X GET "https://api.cryptouse.com/api/onchain/order/pmt_12345abcde" \
  -H "X-App-ID: sua_chave_de_api"
```

### Resposta

<ResponseField name="order" type="object">
  <Expandable title="Propriedades">
    <ResponseField name="value" type="number">
      O valor em BRL da ordem
    </ResponseField>
    <ResponseField name="cryptoValue" type="number">
      O valor em criptomoeda
    </ResponseField>
    <ResponseField name="status" type="string">
      Status atual do pagamento
      
      Valores possíveis: `PENDING`, `COMPLETED`, `REFUNDED`, `PARTIAL_ORDER`, `FAILED`
    </ResponseField>
    <ResponseField name="paymentLinkId" type="string">
      Identificador único para o pagamento
    </ResponseField>
    <ResponseField name="address" type="string">
      Endereço da carteira de criptomoeda
    </ResponseField>
    <ResponseField name="paymentURI" type="string">
      URI de pagamento para a transação
    </ResponseField>
    <ResponseField name="receivedValue" type="number">
      Valor recebido (se pagamento parcial)
    </ResponseField>
    <ResponseField name="originalCryptoValue" type="number">
      Valor original em criptomoeda
    </ResponseField>
    <ResponseField name="receivedFrom" type="string">
      Endereço que enviou o pagamento
    </ResponseField>
    <ResponseField name="partialUpdatedAt" type="string" format="date-time">
      Timestamp de quando o pagamento parcial foi recebido
    </ResponseField>
    <ResponseField name="createdAt" type="string" format="date-time">
      Timestamp de criação
    </ResponseField>
    <ResponseField name="metadata" type="object">
      <Expandable title="Propriedades">
        <ResponseField name="orderId" type="string">
          ID interno da ordem
        </ResponseField>
        <ResponseField name="tokenAddress" type="string">
          Endereço do contrato do token
        </ResponseField>
        <ResponseField name="network" type="string">
          Rede blockchain
        </ResponseField>
        <ResponseField name="walletIndex" type="number">
          Índice da carteira utilizada
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="asset" type="string">
      Tipo de criptomoeda
    </ResponseField>
    <ResponseField name="network" type="string">
      Rede blockchain utilizada
    </ResponseField>
  </Expandable>
</ResponseField>

## Pagamentos parciais

Quando um pagamento parcial é recebido para uma ordem, o status da ordem muda para `PARTIAL_ORDER` e os campos `receivedValue` e `partialUpdatedAt` são preenchidos com os detalhes do pagamento parcial.

Para ordens com pagamentos parciais, você pode calcular o valor restante a ser pago subtraindo `receivedValue` de `cryptoValue`.

## Monitoramento de ordens

Além de consultar ordens manualmente, você pode configurar webhooks para receber notificações automáticas quando o status de uma ordem mudar. Veja a seção [Webhooks](/essentials/webhooks) para mais detalhes.
